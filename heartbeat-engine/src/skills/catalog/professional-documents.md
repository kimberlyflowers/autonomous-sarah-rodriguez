# Professional Documents

## How to Think About This

A professional document is a trust signal. When someone opens a report, reads a proposal, or receives a letter — the formatting, structure, and voice tell them whether this organization is serious.

A sloppy document says "we threw this together." A polished one says "we respect your time and intelligence."

Before creating any document, check memory and Company Skills for the client's brand colors, logo usage, preferred sign-off, and any formatting guidelines.

## Document Types

### Quarterly/Impact Report
**Audience:** Donors, board members, partners, stakeholders
**Key principle:** Data + Story. Numbers alone are a spreadsheet. Stories alone are a newsletter. Together, they're compelling.

Structure: Cover page → Table of Contents → Executive Summary (lead with human impact, not numbers) → Key Metrics table → Program sections (each a mini-story with data) → Financial overview (transparent, show challenges too) → Looking ahead (specific, measurable) → Personal note from leadership.

### Proposal / Partnership Pitch
**Audience:** Potential partners, funders, sponsors
**Key principle:** Frame as THEIR opportunity, not your need. "Here's what you gain" > "Here's what we need."

### Thank-You / Donor Letter
**Audience:** Supporters, donors, clients
**Key principle:** Be specific about impact. "$500 funded [specific thing]" > "Your generous gift supports our mission."

### SOP (Standard Operating Procedure)
**Audience:** Staff, volunteers, AI agents
**Key principle:** Write for someone doing this for the first time. Every step so clear they can't do it wrong.

## Writing Voice

- **Professional but human.** Not robotic corporate speak. Not casual texting. How a thoughtful leader speaks to a respected peer.
- **Honest about challenges.** Transparency builds more trust than pretending everything was perfect.
- **Specific over generic.** Always. Concrete numbers and real examples over vague claims.
- **Close with warmth.** Use the client's preferred sign-off from their Company Skills. Default to something warm like "In service," or "With gratitude," — not "Best regards."

## Formatting Standards

### Layout
- US Letter (8.5" x 11"), 1-inch margins
- Page numbers in footer
- Header: organization name + document title (subtle)
- Single column body text

### Typography
- Headings: Bold, dark color, clear hierarchy
- Body: 11pt, readable font
- Two fonts maximum
- Match client's brand fonts when known

### Tables
- Alternating row shading (very light, never dark)
- Header row: dark background, white text
- Number columns right-aligned or centered
- Always include units

### Colors
- Use client's brand colors when known
- Default: deep navy headings, warm accent color
- Never pure black text (#2D2D2D reads better)
- Super light grey for backgrounds — never cream or eggshell

### Cover Page
- Organization name (large, bold)
- Document title
- Date/period
- Subtle accent element
- "Prepared by" with name and title

## Common Mistakes to Avoid

- Executive summaries that are just a table of contents in paragraph form
- Financial sections that hide bad news
- Generic closings that could be from any organization
- Missing page numbers
- Inconsistent heading styles
- "Looking ahead" with vague goals
- Leadership note that sounds like a press release instead of a person
- No cover page on formal documents
- Decorative borders or clip-art
- Ignoring the client's brand standards

## Output

Always deliver as a formatted file (docx or PDF) when possible, not just markdown. The formatting IS part of the deliverable.
