# Website & Landing Page Design — Premium Quality Standard

This skill produces websites that look like they cost thousands to build. Every page must feel hand-crafted by a top design agency — not generated by AI. The bar is: would a client proudly share this URL with their audience?

## Core Philosophy

Sarah doesn't build "AI websites." She builds **premium digital experiences** that happen to be created fast. The output quality must be indistinguishable from a $5,000-10,000 agency project. Obsessive attention to visual detail, purposeful motion, and conversion psychology.

---

## Phase 1: Understand Before You Design

### Conversion Thinking (mandatory)
Answer before writing any code:
1. **Who lands here?** Where from? What do they know/feel?
2. **What's the ONE action?** Book, buy, sign up, download. ONE.
3. **What's their biggest objection?** Cost? Trust? Time? Address it head-on.

Use Brand Kit for colors/fonts/voice. Use Company Skills for audience/industry context.

### Design Direction
Commit to a BOLD aesthetic:
- Audience → design personality. Parents=warm. Entrepreneurs=bold. Luxury=refined.
- Pick an emotion: Excitement? Calm confidence? Urgency? Safety?
- Don't make "a nice website." Make one that makes people say "wow."

## Phase 2: Visual Design System (Encoded Preferences)

### Typography (#1 differentiator between cheap and premium)
- **ALWAYS use Google Fonts** via CDN `<link>` in `<head>`
- **Heading fonts** (distinctive, never generic):
  Elegant: Playfair Display, Cormorant Garamond, DM Serif Display
  Modern: Plus Jakarta Sans, Space Grotesk, Outfit, Sora
  Bold: Clash Display, Cabinet Grotesk, General Sans
  Friendly: Nunito, Quicksand, Comfortaa
- **Body fonts** (clean, readable): Inter, DM Sans, Source Sans 3, Nunito Sans, Lato
- **Sizes** (responsive): Hero `clamp(2.5rem,5vw,4.5rem)`, sections `clamp(1.75rem,3vw,2.75rem)`, body `1.05-1.15rem` with `line-height:1.7`
- **Never Arial, Helvetica, Times New Roman, or system defaults**
- **Letter-spacing on headings**: `-0.02em` (tighter = premium)
- **Font-weight contrast**: Headlines 700-800, body 400

### Color System
- **FIRST: Check if Brand Kit is in the system prompt.** If yes, use those EXACT colors as your CSS variables:
  ```css
  :root { --primary: [brand kit color 1]; --accent: [brand kit color 2]; --dark: [brand kit color 3]; --light: [brand kit color 4]; }
  ```
- **If Brand Kit fonts are specified**, load those from Google Fonts instead of picking your own.
- **If Brand Kit voice is specified**, write all page copy in that tone.
- **If NO Brand Kit**, create a cohesive 5-color palette: Primary, Accent, Dark, Light, Subtle
- **Gradients**: `linear-gradient(135deg, ...)` on hero backgrounds and CTAs
- **Never pure black** for text → use `#111827` or `#0f172a`
- **Never pure white** for backgrounds → use `#fafafa` or `#f8f9fa`
- **CTA color**: High contrast. Orange/coral outperforms blue/green by 21% in A/B tests
- **Section alternation**: Alternate white ↔ light grey for visual rhythm

### Layout & Spacing
- **Max width**: `1200px` centered
- **Section padding**: `clamp(60px,8vw,120px) clamp(20px,5vw,80px)` — generous
- **White space = premium**. When in doubt, add more padding.
- **Feature grid**: `display:grid; grid-template-columns:repeat(auto-fit,minmax(300px,1fr)); gap:2rem`
- **Z-pattern** for landing pages, **F-pattern** for content pages

### Hero Section (the most important 5 seconds)
- `min-height: 100vh` or `80vh` minimum
- **Value prop in under 10 words** — a promise, not a description
- **Sub-headline**: 1-2 sentences expanding value, addressing pain point
- **Single CTA button**: Large (`padding:16px 40px`), rounded (`border-radius:12px`), gradient, white text, shadow
- **Social proof near CTA**: "Trusted by 500+ families" / star rating — within eyeshot of button
- **Hero visual**: Full-width background with gradient overlay, or split layout (image right, text left)
- **Write REAL copy** based on the client — never placeholder text

### Animations (include ALL of these by default)

#### Scroll-triggered fade-up (every section below fold):
```css
.animate-on-scroll { opacity:0; transform:translateY(30px); transition:opacity 0.6s ease, transform 0.6s ease; }
.animate-on-scroll.visible { opacity:1; transform:translateY(0); }
```
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(e => { if(e.isIntersecting){ e.target.classList.add('visible'); observer.unobserve(e.target); }});
}, { threshold:0.1, rootMargin:'0px 0px -50px 0px' });
document.querySelectorAll('.animate-on-scroll').forEach(el => observer.observe(el));
```

#### Staggered child animations:
```css
.animate-on-scroll.visible:nth-child(1){transition-delay:0s}
.animate-on-scroll.visible:nth-child(2){transition-delay:0.15s}
.animate-on-scroll.visible:nth-child(3){transition-delay:0.3s}
.animate-on-scroll.visible:nth-child(4){transition-delay:0.45s}
```

### Micro-interactions (include by default)
```css
/* Button hover — lift + shadow */
.cta-btn { transition:all 0.3s ease; }
.cta-btn:hover { transform:translateY(-2px) scale(1.02); box-shadow:0 10px 30px rgba(0,0,0,0.15); }

/* Card hover — lift */
.card { transition:transform 0.3s ease, box-shadow 0.3s ease; border-radius:16px; }
.card:hover { transform:translateY(-6px); box-shadow:0 20px 40px rgba(0,0,0,0.08); }

/* Image hover — zoom inside container */
.img-container { overflow:hidden; border-radius:16px; }
.img-container img { transition:transform 0.5s ease; }
.img-container:hover img { transform:scale(1.05); }
```

### Background Effects
```css
/* Gradient blob behind hero */
.hero { position:relative; overflow:hidden; }
.hero::before {
  content:''; position:absolute; width:600px; height:600px;
  background:radial-gradient(circle, rgba(244,162,97,0.15) 0%, transparent 70%);
  top:-200px; right:-100px; border-radius:50%; z-index:0;
}
.hero::after {
  content:''; position:absolute; width:400px; height:400px;
  background:radial-gradient(circle, rgba(231,111,139,0.1) 0%, transparent 70%);
  bottom:-100px; left:-150px; border-radius:50%; z-index:0;
}
```

### Sticky Navigation (always include)
```css
nav { position:fixed; top:0; width:100%; z-index:1000; transition:all 0.3s; padding:16px 0; }
nav.scrolled { background:rgba(255,255,255,0.9); backdrop-filter:blur(20px); box-shadow:0 1px 20px rgba(0,0,0,0.05); padding:10px 0; }
```
```javascript
window.addEventListener('scroll', () => {
  document.querySelector('nav').classList.toggle('scrolled', window.scrollY > 50);
});
```

### Component Standards

**Testimonial cards**: Photo (64px circle), name, title, ⭐⭐⭐⭐⭐, quote. Place NEAR the CTA.

**Feature/benefit cards**: Emoji/SVG icon + heading + 2-line description. 3-column grid desktop, 1-column mobile. `border-radius:16px`, subtle shadow, `padding:24-32px`.

**Footer**: Multi-column (Logo+tagline | Links | Contact | Social). Slightly darker background. Copyright + privacy links at bottom.

### Images
- TRY `image_generate` BEFORE building HTML for hero/section images
- If images generate successfully, use returned `image_url`: `<img src="/api/files/preview/art_xxx">`
- **IF IMAGE GENERATION FAILS** (no API key, error, timeout): DO NOT ABANDON THE BUILD.
  Instead, use beautiful CSS gradient/pattern backgrounds as visual replacements:
  ```css
  /* Hero without image — gradient + pattern fallback */
  .hero { background: linear-gradient(135deg, var(--primary) 0%, var(--accent) 100%); }
  /* Feature card without image — subtle pattern */
  .card-visual { background: linear-gradient(135deg, #f0f0f0 25%, transparent 25%, transparent 50%, #f0f0f0 50%, #f0f0f0 75%, transparent 75%); background-size: 20px 20px; }
  ```
- The website MUST be delivered complete and beautiful regardless of whether images generated
- NEVER abandon a build because one tool failed. Work around it and deliver.
- NEVER use placeholder URLs (placeholder.com, placehold.it) or broken links
- NEVER embed base64 inline — it breaks layouts
- Hero: 1536x1024 landscape. Features: 1024x1024 square.
- All images: `alt` text, `object-fit:cover`, `border-radius:16px`, `loading="lazy"`

### Mobile (non-negotiable)
- Mobile-first design. Breakpoint at `768px`
- `<meta name="viewport" content="width=device-width, initial-scale=1">`
- Touch targets min 44x44px. Font sizes use `clamp()`. Grid → single column.
- Hamburger nav on mobile. No horizontal scroll — ever.

### Performance
- Single HTML file. All CSS inline in `<style>`. Google Fonts via CDN with `display=swap`.
- Lazy load below-fold images. Minimal vanilla JS only (no jQuery, no Bootstrap).

## Phase 3: Build & Deliver

1. Generate images with `image_generate`
2. Build the HTML with all encoded preferences applied
3. Save with `create_artifact` as `.html` file
4. The client can preview in the editor, edit visually, and publish

### Required page structure:
1. `<nav>` — Sticky + blur on scroll + mobile hamburger
2. `<section class="hero">` — Full viewport, value prop, CTA, social proof
3. `<section class="features">` — Benefits grid with icons
4. `<section class="about">` or `<section class="story">` — Who/why section
5. `<section class="testimonials">` — Social proof cards
6. `<section class="cta-final">` — Final call to action
7. `<footer>` — Multi-column links + social + copyright

## Quality Checklist (verify before saving)

- [ ] Hero value prop under 10 words
- [ ] Single clear CTA (not multiple competing)
- [ ] Social proof near CTA
- [ ] Scroll animations on all below-fold sections
- [ ] Button + card hover effects
- [ ] Sticky nav with blur
- [ ] Mobile responsive at 375px
- [ ] Google Fonts (not system fonts)
- [ ] CSS variables for colors
- [ ] Generous white space (80-120px section padding)
- [ ] Complete footer
- [ ] Looks like it cost $5,000+

## NEVER do these:
- Nav links on single-page landing pages (exits)
- "Submit" / "Click Here" button text → use "Get Started", "Book Now", "Claim Your Spot"
- Walls of text without breaks
- Centered body text (fine for headlines only)
- Body text under 16px
- No hover effects
- Missing mobile design
- System fonts
- Pure black on pure white
- Missing viewport meta
- Template-looking AI output
